What is chorgly?

chorgly tracks chores across multiple people. It keeps a shared list of what needs doing, who can do it, and when it is due — without nagging or over-engineering.

Default flow: you open the app, see what is pending, and tap a button to mark something done or add a new chore. No accounts, no passwords — login is by a daily-renewed token link sent by the admin.

Chore types

One-time

A task that needs to be done once and then disappears from the list.

Recurring — time since last completion

Comes back on the list after a fixed interval from when it was last marked done. Example: vacuum every 7 days from the last vacuum.

Recurring — fixed schedule

Appears at a fixed time regardless of when it was last done. Example: take out bins every Sunday evening.

Deadline-driven

Has a hard deadline. Overdue items are highlighted.

Dependencies

A chore can depend on another chore or on an external event. It stays locked until its dependency is satisfied.

Users and access

Some chores are open to everyone; others are restricted to specific users. The app enforces this — you only see chores you are allowed to act on.

There are no passwords. The admin creates a user with a terminal script and sends an init link. The token renews daily and expires after a week of inactivity. Lost tokens are reset by the admin.

Tech stack

Frontend

Rust compiled to WebAssembly, with a thin JS glue layer and a dedicated CSS file. The page renders in basic mode even without JS.

Backend

WebSocket interface. Data stored as CBOR, kept in memory, snapshotted to a git repo every hour when changes occurred. Rate-limited per client. All input validated before any operation.

Auth

Token-based. No passwords. Tokens renewed daily, expire in a week. Created and reset by admin terminal script.
